# TaskType Workflow Definitions v1.0.0

## Purpose

Define standardized workflows for each TaskType, specifying:
- Which Skills are invoked in what order
- Color tags and risk levels for visual reference
- Role and expected output of each step

This document serves as the **single source of truth** for pm-orchestrator workflow execution.

## TaskType Overview

| TaskType | Description | Risk | Typical Duration |
|----------|-------------|------|------------------|
| READ_INFO | Information retrieval, questions | Low | Short |
| LIGHT_EDIT | Single file, minor change | Low | Short |
| IMPLEMENTATION | Multi-file implementation | Medium-High | Long |
| REVIEW_RESPONSE | PR review response | Medium | Medium |
| CONFIG_CI_CHANGE | Configuration/CI changes | Medium | Medium |
| DANGEROUS_OP | Destructive operations | High | Variable |

### User Confirmation Requirements

Before executing DANGEROUS_OP, pm-orchestrator MUST:

1. **Explain Impact Scope**
   - What will be affected
   - Which files/systems will change
   - Potential data loss

2. **Present Rollback Method**
   - How to undo the operation
   - Backup recommendations

3. **Wait for Explicit Approval**
   - User must type specific confirmation phrase
   - "yes", "proceed", "confirm" etc.
